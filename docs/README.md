# 프로젝트 문서 가이드 (Project Documentation Guide)

이 디렉토리는 프로젝트의 모든 기술 문서, 설계안, 그리고 현재 진행 상황을 체계적으로 관리하기 위한 공간입니다.

## 📂 디렉토리 구조 및 역할

### 1. [memory/](./memory/) (AI 워킹 메모리)

- **역할:** AI 에이전트가 작업을 시작할 때 가장 먼저 읽고 현재 상황을 파악하는 핵심 공간입니다.
- **주요 파일:**
  - `project-context.md`: 프로젝트의 헌법 (기술 스택, 아키텍처 원칙, 코딩 컨벤션).
  - `project-status.md`: 현재 진행 상황 (로드맵, 현재 단계의 체크리스트, 알려진 이슈).

### 2. [plans/](./plans/) (작업 설계)

- **역할:** 브레인스토밍을 통해 결정된 새로운 기능의 상세 설계 및 구현 계획이 저장됩니다.
- **규칙:** 작업 진행 중에는 여기에 위치하며, 작업이 완료되어 Branch가 병합되면 `archive/completed-plans/`로 이동합니다.

### 3. [specs/](./specs/) (요구사항 및 명세)

- **역할:** 프로젝트의 비즈니스 요구사항 및 API 상세 명세를 관리합니다.
- **주요 파일:**
  - `api/`: Swagger/OpenAPI 관련 파일 및 API 상세 명세.
  - `stepN-topic.md`: 각 마일스톤별 기능 요구사항 정의서.

### 4. [archive/](./archive/) (기록 보관소)

- **역할:** 구현이 완료된 문서나 과거의 기록을 보관하여 `docs/` 루트를 깔끔하게 유지합니다.
- **주요 파일:**
  - `completed-plans/`: 구현이 끝난 설계 문서 (`plans/`에서 이동됨).
  - `internal-flows/`: 시스템 내부 동작 흐름도 및 상세 로직 설명서.

### 5. [guides/](./guides/) (개발 가이드)

- **역할:** 개발 환경 설정, 툴 사용법 등 협업에 필요한 전반적인 가이드를 제공합니다.

---

## 📝 관리 규칙

1.  **AI 지침:** 모든 새로운 기능 개발은 `brainstorming` 스킬을 통해 `plans/`에 설계 문서를 작성하는 것부터 시작합니다.
2.  **문서 이동:** 작업이 완료되어 `main` 브랜치에 반영될 때, 관련 설계 문서는 `archive/completed-plans/`로 이동시켜 현재 작업 중인 문서와 구분합니다.
3.  **최신화:** 기술 스택이나 컨벤션이 바뀌면 `memory/project-context.md`를, 특정 단계가 끝나면 `memory/project-status.md`를 즉시 업데이트합니다.
